##*Problematica*  
Lo que se quiere resolver con este trabajo es un sistema de gestion estudiantil centralizado que permita modernizar y optimizar el manejo de informacion academica. Este sistema lleva un registro unificado de los estudiantes, sus datos personales, informacion de contacto, y estado academico; Tambien cuenta con la informacion de los docentes, sus roles y materias en las que participan. 
Se puede administrar la oferta academica de cada carrera, definir planes de estudios y correlatividades. Cuenta con un registro del avance academico del estudiante, materias en curso, aprobadas, regulares y notas finales. Ademas, se pueden asignar roles docentes, materias que dictan y periodo en que lo hacen. Tanto alumnos y docentes pueden consultar en cualquier momento esta informacion. 

##*Usuarios*  
Administradores: Gestionaran datos de los alumnos, docentes, las materias, planes de estudio y correlatividades
Alumnos: Consultar y Modificar informacion personal, incribirse en materias, consultar carreras, planes de estudio y sus correlatividades
Docentes: Consultar y Modificar informacion personal, cargar notas a estudiantes, acceder a listado de estudiantes de sus materias

##*Funcionalidades*  
Actualmente se encuentra implementada la funcion 'Alta profesor', la cual consiste en registra un nuevo profesor al sistema cargando su informacion personal para luego poder asignarlo a las asignaturas correspondientes.
Planeamos agregar las funciones de gestionar informacion personal tanto de docentes como de estudiantes, gestionar las notas finales de cada materia de cada alumno, autenticacion de todos los usuarios mediante nombre de usuario y contrasena

##*Tecnologias Utilizadas*  
Utilizamos markdown para crear la documentacion y llevar registro de todo lo que vamos realizando.
Utilizamos java para la creacion de la historia de usuario y Spark Java para la creacion de la API REST. Mustache para las vistas.
JDBC para la interacion con la base de datos. JUnit para las pruebas unitarias. MySQL para la creacion de la Base de datos. UML para realizar los diagramas de clase y sus relaciones.
Todo el proyecto estara disponible en github y proximamente se cargara a GitHub Proyects para una mejor gestion de ciertas tareas como el backlog.

##*Organizacion de las Tareas*
